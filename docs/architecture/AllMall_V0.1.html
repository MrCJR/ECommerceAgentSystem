时代背景与破局之道

Industry Context & Breakthrough

🚨 当下电商运营痛点

人力密集与平台内卷：多店铺矩阵、跨平台铺货与日常数据盯盘耗费大量人力资源。
响应滞后：面对实时波动的 ROI 和流量趋势，人工调整往往存在时间差，难以做到 24 小时精准调配。
规则深水区：各平台合规与风控规则频繁更新，依赖人工经验极易踩坑导致限流或封店。

✨ 2025 AI Agent 带来的机与战

空前机遇：大模型能力从“辅助生成 (Copilot)”正式跨越至“自主执行 (Agent)”，具备了复杂动态环境下的感知、推理与决策能力。
关键挑战：完全脱离人工视角的 AI 极易产生“幻觉”与失控操作（如盲目消耗广告预算）；同时，平台规则和风控策略持续演进，要求自动化系统具备更强的合规校验、频率治理与人工兜底机制。

🧱 现有自动化产品的瓶颈

传统 RPA 太脆弱：高度绑定固定的 DOM 结构和坐标，前端 UI 稍作改版即全面宕机。
API 接口受限：各大平台不断收缩官方 API 权限，高阶运营动作往往无接口可调。
单体架构难扩展：市面上的工具多为单机单账号设计，缺乏底层多租户隔离与全局中控，无法支撑企业级矩阵化运营。

💡 AllMall 的优化与解答

视觉驱动执行：依托 Claw Runner / OpenClaw-compatible 执行引擎结合大模型视觉流与 Playwright，在官方 API 不覆盖的场景下补齐运营动作，降低对固定 DOM 与坐标脚本的依赖。
四层架构风控：将全局大脑、合规网关与执行环境解耦，内置 Human-in-the-Loop (HITL) 熔断机制，兼顾效率、审计与安全边界。
云端隔离：采用租户级数据边界、店铺级会话隔离与独立资源配额，为企业矩阵运营提供稳定、可审计、可回滚的执行环境。

平台工程化控制面

Task · Approval · Audit

统一任务生命周期

  • Task：用户或系统触发的业务目标，如调价、上架、素材替换、用户召回。
  • Workflow：将业务目标拆成可复用流程模板，沉淀平台级 SOP。
  • Run / Step：记录每次执行实例与每一步状态，支持暂停、重试、回滚与补偿。
  • Queue：按租户、店铺、优先级和风险等级进行调度，避免任务互相抢资源。

HITL 审批控制面

  • 审批触发：大额预算调整、批量改价、批量发券、商品下架、违规风险内容。
  • 审批通道：Portal、飞书、钉钉、企业微信均可作为一键确认入口。
  • 审批策略：支持租户级阈值、角色级权限、超时自动取消与二次确认。
  • 审批留痕：保存审批人、审批时间、变更前后值与 Agent 推荐理由。

可观测与可回放

  • 视觉证据：关键步骤截图、页面状态摘要、浏览器 Trace 与异常截图。
  • 决策证据：RAG 命中文档、模型输入输出摘要、风险评分与策略版本。
  • 执行证据：点击、输入、提交、接口调用、失败重试与最终结果统一入审计日志。
  • 事故复盘：支持按租户、店铺、任务、Agent、时间线快速检索。

多租户边界设计

Data · Secret · Quota

数据隔离

  • 企业版优先采用独立数据库或独立 Schema，标准版可采用共享库 + tenant_id + 行级策略。
  • 订单、商品、广告、客户、素材、任务、审计日志均必须带租户与店铺归属。
  • 跨租户统计只进入脱敏后的平台运营视图，不暴露原始业务数据。

密钥隔离

  • 店铺授权 Token、Cookie、平台账号、Webhook、LLM Key、代理配置进入独立 Secret Vault。
  • Worker 只在执行窗口内获取最小权限凭证，任务结束后释放或轮换。
  • 所有敏感配置访问进入审计日志，并支持租户主动吊销。

资源配额

  • 按套餐限制 Worker 数量、并发任务、LLM Token、素材存储、浏览器会话和每日操作次数。
  • 预算型动作必须绑定租户级、店铺级、活动级上限，超过阈值自动进入审批。
  • 异常租户可被限流、暂停 Worker 或进入人工复核模式。

MVP 落地路线

Build Order

Phase 1：平台底座

  • 租户、用户、角色、店铺授权、Secret Vault、资源配额与计费基础。
  • 统一任务中心：Task / Workflow / Run / Step / Approval / Audit Log。
  • MuleRun 内测接入：connectToken 绑定、店铺级浏览器会话、截图与执行日志回传。

Phase 2：首个高价值 Agent

  • 优先选择广告监控或商品上架 Agent，业务闭环清晰、ROI 容易验证。
  • 接入策略阈值、合规校验、异常熔断与人工审批。
  • 沉淀可复用 SOP 模板，为后续 CRM、财务、素材 Agent 复用。

Phase 3：规模化运营

  • 扩展多平台适配层，抽象平台差异：淘宝、抖音、京东、TikTok Shop、Amazon 等。
  • 引入策略实验、A/B 测试、自动报表、租户健康分和成本看板。
  • 建立运维控制台，支持 Claw Runner 演进、Worker 伸缩、失败重放、任务暂停与租户级风控。
