AllMall SaaS-Agent API · V0.1

SaaS 与 Agent 接口文档

本文档定义 AllMall SaaS 与 MuleRun / Agent 在第一阶段联调中的接口契约，重点覆盖店铺登录绑定与竞品调研任务。

Login Bootstrap Competitor Research Runtime Event Approval

1. 接口目录

# 场景 方向 Method Path 说明 详情
1 启动店铺登录绑定 Portal -> SaaS POST /api/stores/{storeId}/login-session 用户点击“登录并绑定店铺”后创建登录会话，返回 MuleRun 沙箱 URL 或手动启动信息。 查看

2.4 Store Authorization 状态

状态说明
disconnected店铺未绑定 runtime session。

revoked用户或管理员已吊销授权。
failed登录绑定失败。

3. 登录绑定流程

登录绑定发生在用户已经登录 AllMall，并且已经创建某个电商平台店铺之后。用户不会在 AllMall 页面输入电商平台账号密码，而是在 MuleRun 浏览器沙箱中完成真实平台登录。AllMall 只负责创建绑定会话、展示绑定入口、接收 Agent 回调，并保存数值型 runtimeSessionId 与外部 runtimeSessionRef。

3.1 普通用户入口

店铺状态AllMall 按钮点击后动作
disconnected 登录并绑定店铺 创建登录绑定会话，进入 MuleRun 登录流程。
login_required 重新登录 重新生成绑定码或沙箱 URL，要求用户重新完成平台登录。
connected 重新绑定 / 断开连接 用户主动重绑或吊销当前 session。

3.2 启动模式

模式用户体验SaaS 返回适用阶段
manual_mulerun_agent 用户在 AllMall 点击“登录并绑定店铺”，复制绑定码，手动打开 MuleRun Login Bootstrap Agent 页面，并在 Agent 页面输入绑定码。 connectToken, agentEntryUrl, manualInstruction Beta 阶段
auto_sandbox_url 用户在 AllMall 点击“登录并绑定店铺”，AllMall 自动打开 MuleRun 浏览器沙箱新标签页。 connectToken, sandboxUrl 产品化阶段

3.3 Beta 手动启动流程

步骤执行方动作结果
1用户在 AllMall 店铺详情页点击 登录并绑定店铺。SaaS 创建登录绑定会话。
2AllMall生成 connectToken，状态变为 pending_login。页面展示绑定码、有效期和 MuleRun Agent 入口。
3用户手动打开 MuleRun Login Bootstrap Agent 页面。进入 Agent 启动页。
4用户在 Agent 页面输入 connectToken。Agent 可向 AllMall 解析绑定上下文。
5Agent调用 /api/agent/connect-tokens/resolve。获取 tenantId、storeId、platform 和 storeName。
6MuleRun创建或加载该店铺专属 browser profile，并打开电商平台登录页。用户进入隔离浏览器沙箱。
7用户在沙箱中输入电商平台账号密码，并完成短信、扫码、滑块等校验。平台登录成功或失败。
8Agent判断登录结果，截图留证，回调 /api/agent/session-bindings。AllMall 保存 session 引用，店铺状态变为 connected。

3.4 产品化一键启动流程

步骤执行方动作结果
1用户在 AllMall 店铺详情页点击 登录并绑定店铺。SaaS 创建登录绑定会话。
2AllMall通过 Runtime Adapter 调用 MuleRun Create Session / Create Run API。MuleRun 创建浏览器沙箱。
3MuleRun返回 sandboxUrl。AllMall 前端打开新标签页或弹窗。
4用户在 MuleRun 沙箱中完成电商平台登录和人机校验。MuleRun 持久化 browser profile。
5Agent回调 /api/agent/session-bindings。AllMall 更新为 connected。
6AllMall店铺详情页轮询或通过 WebSocket 刷新状态。用户看到“已连接”。

3.5 Login Bootstrap Agent 标准流程

阶段Agent 行为失败处理
解析绑定码 接收 connectToken，调用 AllMall 解析接口，确认 token 未过期、未消费、用途为 login_bootstrap。 返回 CONNECT_TOKEN_EXPIRED 或 CONNECT_TOKEN_CONSUMED 时停止流程。
准备浏览器沙箱 创建或加载店铺级 browser profile，确保不同店铺 session 隔离。 沙箱创建失败时回传 run_failed 或绑定失败事件。
打开平台登录页 根据 platform 打开对应电商平台登录入口。 页面无法打开时截图并回传失败原因。
人工登录 等待用户输入账号密码，并完成短信、扫码、滑块等平台校验。 超时或校验失败时回传 login_required 或 HUMAN_VERIFICATION_REQUIRED。
登录成功确认 检查店铺后台或登录后首页是否可访问，并截取证据。 无法确认时标记 failed，保留截图。
绑定回调 回传 runtimeSessionRef、runtimeProfileRef、登录证据和摘要。 AllMall 回调失败时 Agent 应按幂等键重试。
电商平台账号、密码、扫码、短信验证码和滑块验证只发生在 MuleRun 浏览器沙箱内。AllMall 不收集、不保存、不转发电商平台密码。

4. 启动店铺登录绑定

POST/api/stores/{storeId}/login-session

用户在 AllMall 店铺详情页点击 登录并绑定店铺 或 重新登录 后调用。

Request Body

{
  "platform": "pinduoduo",
  "purpose": "login_bootstrap",
  "runtimeProvider": "mulerun",
  "preferredLaunchMode": "auto_sandbox_url"
}
字段类型必填说明
platformstring是电商平台，第一阶段为 pinduoduo。
purposestring是固定为 login_bootstrap。
runtimeProviderstring是第一阶段为 mulerun。
preferredLaunchModestring否auto_sandbox_url 或 manual_mulerun_agent。

9. Agent 事件回调

POST/api/agent/events

9.1 Event Types

eventType说明SaaS 处理
run_startedAgent 已接收任务并开始执行。Run -> running
step_started某一步骤开始。Step -> running
step_completed某一步骤完成。Step -> succeeded
approval_required需要人工确认。创建 Approval，Run -> waiting_approval
login_requiredsession 失效或需要人机校验。授权状态 -> login_required
data_uncertain数据口径不确定或样本不足。标记低置信度。
run_failed任务失败。Run -> failed
run_completed任务完成并生成报告。Run / Task -> succeeded

11. Artifact 规范

type用途必要 metadata
screenshot登录证明、页面证据。pageUrl, capturedAt, stepKey
raw_data采集的结构化原始数据。schemaVersion, recordCount, source
log执行日志或错误日志。level, stepKey, traceId
report最终报告或中间报告。reportType, confidence, generatedAt
trace浏览器 trace 或工具调用链。runtimeProvider, runtimeRunRef

12. 错误码

错误码场景建议处理
CONNECT_TOKEN_EXPIRED绑定码过期。重新生成登录绑定会话。
CONNECT_TOKEN_CONSUMED绑定码已使用。拒绝重复绑定，写入审计日志。
SESSION_NOT_FOUNDruntime session 不存在。标记 login_required。
SESSION_EXPIRED电商平台要求重新登录。触发重新登录流程。
HUMAN_VERIFICATION_REQUIRED出现扫码、短信、滑块等校验。暂停执行，提示用户人工处理。
PAGE_STRUCTURE_CHANGED页面结构变化导致采集失败。回传截图和失败步骤。
DATA_UNCERTAIN价格、销量、评价等口径不确定。标记低置信度或触发人工确认。
APPROVAL_TIMEOUT审批超时。Run -> failed 或 cancelled。

13. 安全要求

  • connectToken 必须短期有效、一次性消费，并绑定 tenantId / storeId / purpose / runtimeProvider。
  • Agent 回调必须携带签名、时间戳和幂等键。
  • 所有登录绑定、session 变更、任务事件、artifact、审批决定必须写入 Audit Log。
  • 截图、日志和原始数据 URL 必须使用短期签名访问。
  • 竞品调研第一阶段只做读取、分析、确认和留痕，不执行改价、报名活动、修改商品、删除商品等写操作。
