AllMall API Specification · V0.1

SaaS 平台 API 接口文档

本文档定义 AllMall 第一阶段 SaaS 平台 API 的资源边界和联调契约，重点覆盖店铺授权、connectToken、任务控制面、审批、Agent 回调和审计证据。

1. API 设计原则

  • 所有业务 API 必须在服务端解析用户身份和租户上下文。
  • 前端不得直接调用 MuleRun API，也不得暴露 MuleRun Agent Key。
  • 外部 runtime 回调必须使用签名或短期 token 校验。
  • 接口返回应包含明确的状态字段，避免仅依赖 HTTP 状态码表达业务状态。
  • 高风险动作不直接执行，必须产生审批请求或进入等待状态。

2. 资源接口总览

领域接口示例说明
租户与用户/api/tenants, /api/users, /api/roles公司、成员、角色管理。
店铺/api/stores, /api/store-authorizations店铺创建、授权状态、重新登录、吊销。
connectToken/api/connect-tokens生成和校验 MuleRun 绑定令牌。
任务/api/tasks, /api/runs, /api/steps任务创建、执行时间线、步骤状态。
审批/api/approvals审批列表、批准、拒绝、过期。
Runtime 回调/api/runtime/session-bindings, /api/runtime/eventsMuleRun 或其他 runtime 回调入口。

3. 店铺与登录绑定接口

创建店铺

POST /api/stores

{
  "platform": "tiktok_shop",
  "storeName": "Demo TikTok Store"
}

生成 connectToken

POST /api/stores/{storeId}/connect-token

{
  "purpose": "login_bootstrap",
  "runtimeProvider": "mulerun"
}

MuleRun 回调绑定 session

POST /api/runtime/session-bindings

{
  "connectToken": "arkops_connect_xxx",
  "storeId": "store_001",
  "platform": "tiktok_shop",
  "runtimeProvider": "mulerun",
  "runtimeSessionId": "mr_session_xxx",
  "status": "connected"
}

店铺重新登录

POST /api/stores/{storeId}/reauth-token

{
  "reason": "session_expired",
  "runtimeProvider": "mulerun"
}

4. 任务接口

创建 Agent 任务

POST /api/tasks

{
  "storeId": "store_001",
  "agentType": "ads_optimizer",
  "goal": "分析低 ROI 广告计划并提出预算调整建议",
  "constraints": {
    "maxBudgetChangePercent": 15,
    "requiresApprovalAboveAmount": 500
  }
}

查询任务详情

GET /api/tasks/{taskId}

查询执行时间线

GET /api/runs/{runId}/timeline

5. Agent Runtime 回调接口

事件回调

POST /api/runtime/events

{
  "taskId": "task_123",
  "runId": "run_456",
  "eventType": "step_completed",
  "stepName": "Analyze campaign ROI",
  "status": "succeeded",
  "summary": "发现 3 个广告计划低于目标 ROI。",
  "artifacts": [
    {
      "type": "screenshot",
      "url": "s3://arkops-artifacts/run_456/roi-dashboard.png"
    }
  ]
}

需要人工审批

POST /api/runtime/events

{
  "taskId": "task_123",
  "runId": "run_456",
  "eventType": "approval_required",
  "reason": "预算调整超过租户阈值",
  "proposedAction": {
    "type": "pause_campaign",
    "campaignId": "campaign_102"
  },
  "riskLevel": "high"
}

需要重新登录

POST /api/runtime/events

{
  "taskId": "task_123",
  "runId": "run_456",
  "eventType": "login_required",
  "storeId": "store_001",
  "reason": "TikTok Shop session expired or requires human verification"
}

6. 审批接口

GET /api/approvals?status=pending

POST /api/approvals/{approvalId}/approve
{
  "comment": "同意本次预算调整"
}

POST /api/approvals/{approvalId}/reject
{
  "comment": "暂不调整该广告计划"
}
审批通过后，AllMall 可以通过 runtime adapter 通知 MuleRun 继续执行，或在内测阶段让 Agent 轮询审批状态。
