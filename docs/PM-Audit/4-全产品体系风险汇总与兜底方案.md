# 全产品体系风险汇总与兜底方案：AllMall AI Commerce Ops

> 审计标准：PM-Audit-Skill v3 | 基线版本：2026-07-10-v5 | 范围：全产品体系

---

## 一、业务断点风险

| 编号 | 风险 | 风险等级 | 产品层面兜底方案 |
|------|------|:--:|-----------------|
| R01 | 角色权限未执行 — Viewer 可执行所有操作，Operator 可修改系统配置 | **P0** | 短期：AppShell 添加 roleGuard 路由拦截，Viewer 仅可查看不可操作；Operator 不可访问设置模块。长期：后端 API 鉴权 + 操作审计 |
| R02 | 新手用户无引导 — 首次登录后面对 Dashboard 不知从何开始 | **P1** | 短期：Dashboard 添加首次登录 3 步引导浮层（检查店铺→开通 Agent→查看成果）。长期：产品内嵌智能助手根据用户行为主动推荐下一步操作 |
| R03 | 成员操作无日志 — 无法追溯谁在何时做了什么 | **P1** | 短期：审计日志模块增加成员维度筛选和操作者字段。长期：操作日志自动关联角色和审批链 |
| R04 | 表单无输入校验 — 用户可输入不合理参数导致策略失效 | **P2** | SetupConfigPage 和策略配置组件添加 InputNumber min/max、必填校验、异常值提示。重点：利润率 0-100%、广告预算 > 0 |
| R05 | 语言切换不同步 — 登录页和 AppShell 独立维护语言状态 | **P2** | 统一语言状态到全局 Context，LoginPage 语言切换触发全局刷新 |
| R06 | 无日期对比能力 — 用户无法判断本月表现好坏 | **P2** | Dashboard 添加日期范围选择器和环比数据展示（本月 vs 上月、今年 vs 去年） |
| R07 | 无 Target Setting — 用户无目标和进度感知 | **P2** | Dashboard 添加 GMV/订单量月度目标设定和进度条展示 |
