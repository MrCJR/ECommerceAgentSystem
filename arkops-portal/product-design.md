# AllMall 产品设计文档

## 一、产品概述

### 1.1 产品定位

AllMall 是一个电商店铺自动化运营平台，通过 AI Agent 代替运营人员执行店铺日常操作（广告优化、新品上架、CRM 复购、财务对账等），实现大规模店铺的高效自动化管理。

### 1.2 商业化模式

| 模式 | 部署方式 | 计费方式 | 目标客户 |
|------|---------|---------|---------|
| SaaS 订阅版 | 云端多租户 | 订阅制 + 超额按量付费 | 中小型卖家、代理商 |
| 本地部署版 | 客户自有服务器 / 私有云 | 年付 License + 扩容 | 大型品牌、对数据合规要求高的企业 |

### 1.3 用户角色

| 角色 | 权限范围 |
|------|---------|
| Owner（所有者） | 全部权限，含删除店铺、管理订阅与账单 |
| Admin（管理员） | 管理店铺 / Agent / 任务 / 成员，不可删除团队 |
| Operator（运营） | 管理自己负责的店铺，创建 / 审批任务 |
| Approver（审批人） | 仅审批高风险操作，不可创建任务 |
| Finance（财务） | 仅查看财务台账，管理订阅 |
| Viewer（只读） | 仅查看数据大屏和报表 |

## 三、功能模块详细设计

### 3.1 控制台（Dashboard）

**定位：** 运营负责人打开平台的第一眼，经营数据 + Agent 监控分层展示。

控制台分为两个 Tab：**经营概览**（业务老板视角）和 **Agent 监控**（平台运维视角）。

#### 3.1.1 经营概览 Tab ✅

- **核心指标卡片：** 今日 GMV（环比涨跌）、今日订单数、客单价、店铺数
- **GMV & 订单趋势柱状图：** 近 7 天 GMV 与订单双柱对比
- **广告 ROI 面板：** 广告消耗、ROAS、CPM、CPC、CTR、CVR，含预算进度条
- **广告计划列表：** 各计划花费与 ROI 标签
- **售后面板：** 退货率、差评数/未处理、店铺评分、退货金额、回复率、平均响应时间
- **库存与商品：** SKU 总数、低库存/断货/滞销数
- **TOP 5 商品排名：** 按 GMV 降序，金银铜 Tag 标记
- **店铺 GMV 排名：** 按店铺 GMV 排序

#### 3.1.2 Agent 监控 Tab ✅

- **任务状态分布：** 成功/等待审批/运行中/失败 分色进度条
- **Agent 运行概览：** 各 Agent 运行数 + 彩色柱状图
- **额度使用情况：** 店铺连接/浏览器会话/日操作/Token 用量进度条
- **Runtime 健康信号：** 会话健康度/成功率/审批等待/需重登
- **实时动态流：** 任务开始/完成/审批/店铺连接变更等事件滚动展示

### 3.2 店铺管理（Store Management）

**定位：** 所有自动化操作的起点，店铺连接是一切的基础。

#### 3.2.1 店铺列表 ✅

- **列信息：** 店铺名称→链接到详情、平台、授权方式 Tag、状态 Badge、服务标签（千川⚡/飞鸽💬 等）
- **操作：** 新增店铺

> 与早期设计差异：移除了「执行环境」和「最近验证」技术列，改为直观的服务标签。移除了「生成 connectToken」和「标记重新登录」按钮，仅保留「撤销授权」。

#### 3.2.2 店铺新建 / 入驻流程 ✅

1. 平台选择 + 店铺名称
2. 授权方式（账号密码 / API Key / OAuth）
3. 连接配置

#### 3.2.3 店铺详情页 ✅

两个 Tab：「经营概览」和「店铺设置」。

**经营概览 Tab：**
- 核心指标卡片（彩色左边框）：今日 GMV（含环比）、今日订单、客单价、店铺评分
- GMV & 订单趋势柱状图 + 广告投放面板
- 售后服务面板（退货率/差评数/退货金额/纠纷单）
- 库存面板（SKU数/低库存/断货/滞销 + 低库存清单）
- TOP 5 商品排名

**店铺设置 Tab：**
- 店铺主体授权（平台、授权方式、状态、API Key、账号、地区、货币）
- 服务授权（添加/管理服务连接，如千川/飞鸽等）

### 3.3 Agent中心（AgentCenter）

**定位：** 自动化运营的"发动机"。15 个 AI Agent 覆盖从店铺保活到广告投放、从评价管理到财务对账的完整运营链路。

#### 3.3.1 Agent 体系全貌

| # | Agent | 层级 | 触发模式 | 策略配置 | 审批 | 依赖 | 说明 |
|---|-------|------|:--:|------|:--:|------|------|
| 1 | 店铺保活 | 基础管道 | 事件驱动 | - | - | 无 | 必须启用不可关闭，掉线自动拉起 |
| 2 | 市场情报 | 基础管道 | 定时（每2h） | 监控类目/竞品URL | - | 无 | 竞品数据+市场趋势，为其他 Agent 提供数据燃料 |
| 3 | 商品上架 | 基础管道 | **手动创建** | 类目/目标市场 | 需要 | 市场情报 | 上传图片→AI解析→生成草稿→人工确认→上架 |
| 4 | 素材工厂 | 基础管道 | 事件驱动 | 输出尺寸/文案风格 | 需要 | 上架+情报 | 商品发布后自动生成广告图片/视频/文案 |
| 5 | 广告投放 | 流量引擎 | 定时（每6h） | 投流日/月预算 | 需要 | 上架+情报+素材 | 自动创建/调优/暂停广告计划 |
| 6 | 定价策略 | 流量引擎 | 定时（每4h） | 定价模式/利润率/竞争策略 | 需要 | 市场情报 | 动态调价保持竞争力 |
| 7 | 直播运营 | 流量引擎 | 事件驱动 | 置顶策略/回复模板/预警阈值 | - | 商品上架 | 直播中自动置顶热销品、回复评论、监控数据 |
| 8 | CRM 复购 | 增值运营 | 定时（每周一） | 折扣上限/分群数 | - | 无 | 自动发券触达沉默客户 |
| 9 | 评价管理 | 增值运营 | 定时（每2h） | 自动回复星级/回复语气 | - | 无 | 监控差评自动生成回复，标记敏感评价 |
| 10 | 客服消息 | 增值运营 | 事件驱动 | 自动回复开关/转人工关键词 | - | 无 | 自动回复售前咨询，复杂问题转人工 |
| 11 | 售后处理 | 增值运营 | 事件驱动 | 自动退款上限/退货地址 | 需要 | 无 | 低于阈值自动退款，超额转审批 |
| 12 | 促销管理 | 增值运营 | **手动创建** | 折扣上限/活动预算/目标平台 | 需要 | 市场情报 | 策划闪购/大促/限时折扣，跨平台同步 |
| 13 | 库存预警 | 支撑服务 | 定时（每4h） | 低库存阈值/滞销天数/自动补货 | - | 无 | 低库存/滞销/断货告警并建议补货 |
| 14 | 风险控制 | 独立服务 | 定时（每1h） | 三层防护配置 | 需要 | 无 | 全链路安全护栏，包裹所有 Agent |
| 15 | 财务对账 | 独立服务 | 定时（每月1日） | 对账日/差异阈值/自动报告 | 需要 | 无 | 拉取平台账单，标记差异，生成报告 |

> **触发模式说明**：
> - **事件驱动**：由业务事件自动触发（店铺掉线、新消息、退货请求、直播开始、商品发布），无需人工干预
> - **定时（Cron）**：按固定周期自动运行，用户只需开启开关即可
> - **手动创建**：需要用户在 Agent 详情页主动创建任务，适用于需要人工输入的上架和促销场景

#### 3.3.2 逐 Agent 详细设计

---

##### Agent #1：店铺保活（login_bootstrap）

| 属性 | 内容 |
|------|------|
| **层级** | 基础管道层 |
| **运行模式** | 🟢 全自动（事件驱动 `store_session_expired`） |
| **策略配置** | 无需配置。仅有一个执行参数「通知渠道」默认飞书+邮件 |
| **审批** | 无需审批 |
| **必须性** | ⚠️ 系统必须 Agent，不可禁用。禁用后依赖它的 Agent 全部失效 |
| **AI 模型** | DeepSeek deepseek-chat |
| **超时/重试** | 10 分钟 / 最多 3 次，间隔 2 分钟 |

**内置工作流：**
- **会话状态巡检**：定时检查所有已连接店铺的登录态是否正常
- **掉线自动拉起**：检测到会话失效 → 自动重建 → 失败则通知运营
- **批量店铺巡检**：按固定周期对所有店铺执行一轮健康检查

**用户场景：** 后台守护 Agent，用户无需关注其运行。Dashboard 健康信号灯反映其状态。唯一可能的人工介入场景：人机验证需要运营手动完成。

**设计评估：** ✅ 设计合理。作为基础设施 Agent，自动化程度高、配置简单。可以在 Dashboard 增加「店铺健康度」展示保活成功率。

##### Agent #2：市场情报（competitor_intel）

| 属性 | 内容 |
|------|------|
| **层级** | 基础管道层（数据燃料角色） |
| **运行模式** | 🟢 全自动（定时 `0 0 */2 * *` = 每 2 小时） |
| **策略配置** | 监控频率 / 监控类目（默认：消费电子、服装、家居）/ 竞品店铺 URL 列表 |
| **审批** | 无需审批 |
| **依赖** | 无（但被 4 个 Agent 依赖：商品上架、广告投放、定价策略、素材工厂） |
| **AI 模型** | DeepSeek deepseek-chat |
| **超时/重试** | 25 分钟 / 最多 2 次，间隔 30 分钟 |

**内置工作流：**
- **被动竞品监控**：持续采集竞品店铺的价格、促销、新品、评分变化
- **商品研究**：分析竞品热销商品的特征（价格带、卖点、主图风格、关键词）
- **市场趋势监控**：追踪品类热词、消费者画像变化、季节性趋势

**用户场景：** 用户在 Agent 详情页配置竞品 URL 和监控类目 → Agent 自动采集分析 → 结果被上架/定价/广告 Agent 消费，生成洞察建议。

**设计评估：** ✅ 设计合理。作用是为其他 Agent 提供数据输入，不建议重复执行参数（URL配置）和策略配置（categories）的存储位置 — 当前 strategyConfig 和 executionParams 各存一份，应统一到 strategyConfig 中。

##### Agent #3：商品上架（product_launch）

| 属性 | 内容 |
|------|------|
| **层级** | 基础管道层 |
| **运行模式** | 🔴 **需手动创建任务**（triggerMode: `manual`） |
| **策略配置** | 默认商品类目 / 目标市场（默认 US） |
| **审批** | 需要审批（Operator 角色），高风险操作需 Approver |
| **依赖** | 市场情报（必须先开通） |
| **AI 模型** | Anthropic Claude Sonnet 4 |
| **超时/重试** | 20 分钟 / 最多 1 次，间隔 3 分钟 |

**内置工作流：**
- **图片识别（人工触发）**：用户上传商品图片 → AI 解析商品信息（品类/颜色/材质/卖点）
- **草稿自动生成（自动）**：基于识别结果和历史数据，生成 SEO 标题、卖点文案、详情描述
- **合规检查（自动）**：上架前自动校验类目匹配、违禁词、图片合规（若风控开启）

**完整用户任务流程：**
```
用户点击「新建任务」→ 上传商品图片（1-5 张）
  → AI 解析商品属性 → 展示识别结果（品类/颜色/材质/尺寸等，用户可修正）
  → AI 生成商品草稿（标题/卖点/描述/价格建议）→ 用户预览编辑
  → 提交 → 审批（Operator）→ 审批通过 → 自动上架
```

> 商品上架和素材工厂的区别：上架 Agent 负责**商品信息到店铺后台**的环节；素材工厂负责从商品图片**生成广告投放用素材**。依赖关系上，素材工厂依赖商品上架（等商品发布了再生成素材）。

**设计评估：** ⚠️ 需优化。当前表格标注「定价规则」为配置项，但 strategyConfig 中没有 pricingRule — 定价策略是独立 Agent（pricing_strategy）。应改为「默认商品类目/目标市场」。另外 product_launch 没有独立的 strategyConfig 对象，参数散落在 executionParams 中，建议统一。

##### Agent #4：素材工厂（creative_factory）

| 属性 | 内容 |
|------|------|
| **层级** | 基础管道层 |
| **运行模式** | 🟢 全自动（事件驱动 `product_published`） |
| **策略配置** | 输出尺寸（默认 1:1, 16:9, 9:16）/ 文案风格（默认「简洁卖点」） |
| **审批** | 需要审批（Operator 角色） |
| **依赖** | 商品上架 + 市场情报 |
| **AI 模型** | Anthropic Claude Sonnet 4 |
| **超时/重试** | 30 分钟 / 最多 2 次，间隔 5 分钟 |

**内置工作流：**
- **图片生成（自动）**：商品发布后自动生成多尺寸广告主图（1:1 信息流 / 16:9 视频封面 / 9:16 短视频）
- **视频生成（自动）**：从商品图片生成轮播短视频，适配 TikTok/Reels 格式
- **文案生成（定时）**：结合市场情报的竞品分析，生成不同风格的广告文案

**用户场景：** 商品上架 Agent 发布商品 → 自动触发素材工厂 → 生成多套广告素材 → 用户预览 → 通过审批 → 素材进入广告投放 Agent 可用池。

**设计评估：** ✅ 设计合理，但有一个问题：素材工厂标记为「需审批」但生成的素材量可能很大，逐个审批不现实。建议改为「批量预览+选择性驳回」模式，默认通过、异常标记。

##### Agent #5：广告投放（ads_optimizer）

| 属性 | 内容 |
|------|------|

#### 3.3.3 风控体系

风险控制 Agent 开启后作为全链路安全护栏，所有 Agent 操作执行前通过三层检测：

| 层级 | 项目 | 类型 | 说明 |
|------|------|------|------|
| **法规合规** | 广告法禁用词过滤 | Switch | 商品标题/描述/广告文案含违禁词自动拦截 |
| | 平台规则合规检测 | Switch | 各平台（TikTok/Amazon/Shopee）特有规则校验 |
| | 虚假宣传检测 | Switch | "第一""最好""绝对"等夸大用语识别 |
| **行为监控** | ROI 红线 | Input | ROI 低于阈值自动暂停广告 |
| | 操作频率限制 | Input | 单 Agent 每分钟最大操作次数 |
| | 价格异常偏差 | Input | 价格突变超过偏差%触发告警 |
| **业务保护** | 最低售价门槛 | Input | 售价不得低于 成本×ratio |
| | 类目匹配检测 | Switch | 上架时校验类目是否正确 |
| | 图片合规检测 | Switch | 检测水印/Logo/版权元素 |
| | 库存透支保护 | Switch | 库存不足时阻止继续投流 |
| | 差评激增预警 | Switch | 差评数突增自动触发审批 |

> 风控 Agent 未开启时，其他 Agent 正常运行，无安全护栏。**建议在首次开通广告投放/定价策略等高风险 Agent 时，弹窗强提醒用户开启风控。**

```
    │
    ├──→ CRM 复购（无依赖）
    ├──→ 评价管理（无依赖）
    ├──→ 客服消息（无依赖）
    ├──→ 售后处理（无依赖）
    ├──→ 库存预警（无依赖）
    ├──→ 财务对账（无依赖）
    │
    └──→ 风险控制（全链路安全护栏，可选但强烈建议开启）
```

> 开通依赖未满足的 Agent 时，卡片会红字提示缺少的依赖，且不显示开通按钮。

#### 3.3.5 Agent 列表页 ✅

- **已开通区：** 已启用的 Agent 卡片（店铺保活标记红色「必须」Tag）
- **可开通区：** 按 5 层分组（基础管道/流量引擎/增值运营/支撑服务/独立服务），每层有色标
- **卡片信息：** Agent 名称、描述、风险等级 Tag、如需配置显示蓝色「策略配置」Tag、如需审批显示橙色「需要审批」Tag、手动创建显示「手动任务」Tag、依赖未满足显示红色警告
- 依赖满足后显示「开通」按钮

#### 3.3.6 Agent 详情页 ✅

- **页面 Header：** Agent 名称、描述、启用/停用开关（必须 Agent 置灰不可关）
- **运行统计：** 总执行次数、成功率、成功数、失败数
- **运行说明：** 功能描述、触发方式、风险等级、审批要求、常见失败原因、默认参数
- **策略配置：** 按 Agent 类型动态渲染对应配置项（定价规则/投流预算/SEO关键词/人群标签/CRM折扣/退款额度/素材尺寸/库存阈值/风控三层/促销参数/直播参数/对账参数）
- **内置任务工作流：** 以卡片网格展示该 Agent 的核心自动化任务，每种任务有图标、名称、描述和运行标签（自动/定时/被动/人工触发）
- **运行中任务：** 当前进行中/排队中的任务，每行可取消
- **任务日志：** 历史已完成/失败/已取消的任务，显示最近事件摘要
- **新建任务：** 仅在「手动创建」模式的 Agent 显示（商品上架/促销管理）

### 3.4 模型中心（Model Center）

**定位：** AI 能力的"燃料"管理，决定 Agent 的聪明程度和运营成本。

#### 3.4.1 当前架构 ✅

模型中心完全重新设计，去掉了「供应商 → 模型」两级结构和费用统计，改为平铺的「Agent ↔ 模型」分配模式：

- **Agent 模型分配：** 每个 Agent 一行，右侧下拉框直接选择使用的模型。选项来自平台模型 + 我的模型中已激活的。
- **我的模型（自有模型）：** 用户点击「添加模型」→ 下拉选择模型类型（GPT-4o/Claude/DeepSeek/通义千问等 11 种）→ 输入 API Key → 自动创建。自有模型不统计费用。可删除。
- **月度用量统计：** 按月柱状图展示 Agent 调用/Token/浏览器会话/店铺数的月度趋势。

### 3.6 异常中心（Exception Center）

**定位：** 运营人员的优先处理队列，聚合需要人工介入的异常。

- **异常列表：** 登录失效、任务失败、审批超时、数据不确定、订单异常等。
- **优先级分层：** 根据风险等级、影响店铺、等待时间排序。
- **处理动作：** 重新登录、重试任务、查看证据、进入审批、跳转相关店铺或 Agent。
- **侧边栏角标：** 异常中心菜单项后显示待处理异常数量。
