# ArkOps 产品设计文档

## 一、产品概述

### 1.1 产品定位

ArkOps 是一个电商店铺自动化运营平台，通过 AI Agent 代替运营人员执行店铺日常操作（广告优化、新品上架、CRM 复购、财务对账等），实现大规模店铺的高效自动化管理。

### 1.2 商业化模式

| 模式 | 部署方式 | 计费方式 | 目标客户 |
|------|---------|---------|---------|
| SaaS 订阅版 | 云端多租户 | 订阅制 + 超额按量付费 | 中小型卖家、代理商 |
| 本地部署版 | 客户自有服务器 / 私有云 | 年付 License + 扩容 | 大型品牌、对数据合规要求高的企业 |

### 1.3 用户角色

| 角色 | 权限范围 |
|------|---------|
| Owner（所有者） | 全部权限，含删除店铺、修改订阅计划 |
| Admin（管理员） | 管理店铺 / Agent / 任务 / 成员，不可删除团队 |
| Operator（运营） | 管理自己负责的店铺，创建 / 审批任务 |
| Approver（审批人） | 仅审批高风险操作，不可创建任务 |
| Finance（财务） | 仅查看财务台账，管理订阅 |
| Viewer（只读） | 仅查看数据大屏和报表 |

## 三、功能模块详细设计

### 3.1 控制台（Dashboard）

**定位：** 运营负责人打开平台的第一眼，经营数据 + Agent 监控分层展示。

控制台分为两个 Tab：**经营概览**（业务老板视角）和 **Agent 监控**（平台运维视角）。

#### 3.1.1 经营概览 Tab ✅

- **核心指标卡片：** 今日 GMV（环比涨跌）、今日订单数、客单价、店铺数
- **GMV & 订单趋势柱状图：** 近 7 天 GMV 与订单双柱对比
- **广告 ROI 面板：** 广告消耗、ROAS、CPM、CPC、CTR、CVR，含预算进度条
- **广告计划列表：** 各计划花费与 ROI 标签
- **售后面板：** 退货率、差评数/未处理、店铺评分、退货金额、回复率、平均响应时间
- **库存与商品：** SKU 总数、低库存/断货/滞销数
- **TOP 5 商品排名：** 按 GMV 降序，金银铜 Tag 标记
- **店铺 GMV 排名：** 按店铺 GMV 排序

#### 3.1.2 Agent 监控 Tab ✅

- **任务状态分布：** 成功/等待审批/运行中/失败 分色进度条
- **Agent 运行概览：** 各 Agent 运行数 + 彩色柱状图
- **额度使用情况：** 店铺连接/浏览器会话/日操作/Token 用量进度条
- **Runtime 健康信号：** 会话健康度/成功率/审批等待/需重登
- **实时动态流：** 任务开始/完成/审批/店铺连接变更等事件滚动展示

### 3.2 店铺管理（Store Management）

**定位：** 所有自动化操作的起点，店铺连接是一切的基础。

#### 3.2.1 店铺列表 ✅

- **列信息：** 店铺名称→链接到详情、平台、授权方式 Tag、状态 Badge、服务标签（千川⚡/飞鸽💬 等）
- **操作：** 新增店铺

> 与早期设计差异：移除了「执行环境」和「最近验证」技术列，改为直观的服务标签。移除了「生成 connectToken」和「标记重新登录」按钮，仅保留「撤销授权」。

#### 3.2.2 店铺新建 / 入驻流程 ✅

1. 平台选择 + 店铺名称
2. 授权方式（账号密码 / API Key / OAuth）
3. 连接配置

#### 3.2.3 店铺详情页 ✅

两个 Tab：「经营概览」和「店铺设置」。

**经营概览 Tab：**
- 核心指标卡片（彩色左边框）：今日 GMV（含环比）、今日订单、客单价、店铺评分
- GMV & 订单趋势柱状图 + 广告投放面板
- 售后服务面板（退货率/差评数/退货金额/纠纷单）
- 库存面板（SKU数/低库存/断货/滞销 + 低库存清单）
- TOP 5 商品排名

**店铺设置 Tab：**
- 店铺主体授权（平台、授权方式、状态、API Key、账号、地区、货币）
- 服务授权（添加/管理服务连接，如千川/飞鸽等）

### 3.3 Agent中心（AgentCenter）

**定位：** 自动化运营的"发动机"。13 个 Agent 覆盖从店铺保活到广告投放、从评价管理到财务对账的完整运营链路。

#### 3.3.1 Agent 体系全貌

| # | Agent | 层级 | 配置 | 审批 | 依赖 | 说明 |
|---|-------|------|------|------|------|------|
| 1 | 店铺保活 | 基础管道 | - | - | 无 | 必须启用，不可关闭。 |
| 2 | 商品上架 | 基础管道 | 定价规则 | 需要 | 市场情报 | 上传图片→AI解析→生成草稿→上架 |
| 3 | 广告投放 | 流量引擎 | 投流预算 | 需要 | 上架+情报+素材 | 自动创建/调优/暂停广告计划 |
| 4 | 定价策略 | 流量引擎 | 定价规则 | 需要 | 市场情报 | 动态调价 |
| 5 | CRM 复购 | 增值运营 | 折扣+分群 | - | 无 | 自动发券触达沉默客户 |
| 6 | 评价管理 | 增值运营 | - | - | 无 | 监控差评，自动生成回复 |
| 7 | 客服消息 | 增值运营 | - | - | 无 | 自动回复，复杂问题转人工 |
| 8 | 售后处理 | 增值运营 | 退款额度 | 需要 | 无 | 低于阈值自动退款 |
| 9 | 市场情报 | 支撑服务 | SEO/人群 | - | 无 | 竞品数据+市场趋势，底层数据源 |
| 10 | 素材工厂 | 支撑服务 | 尺寸+文案 | 需要 | 上架+情报 | 商品图片→广告素材 |
| 11 | 库存预警 | 支撑服务 | - | - | 无 | 低库存/滞销/断货告警 |
| 12 | 风险控制 | 独立服务 | 三层防护 | 需要 | 无 | 全链路安全护栏 |
| 13 | 财务对账 | 独立服务 | - | 需要 | 无 | 拉取平台账单，标记差异 |

#### 3.3.2 风控体系

风险控制 Agent 开启后作为全链路安全护栏，所有 Agent 操作执行前通过三层检测：

| 层级 | 项目 | 类型 | 说明 |
|------|------|------|------|
| **法规合规** | 广告法禁用词过滤 | Switch | 商品标题/描述/广告文案含违禁词自动拦截 |
| | 平台规则合规检测 | Switch | 各平台（TikTok/Amazon/Shopee）特有规则校验 |
| | 虚假宣传检测 | Switch | "第一""最好""绝对"等夸大用语识别 |
| **行为监控** | ROI 红线 | Input | ROI 低于阈值自动暂停广告 |
| | 操作频率限制 | Input | 单 Agent 每分钟最大操作次数 |
| | 价格异常偏差 | Input | 价格突变超过偏差%触发告警 |
| **业务保护** | 最低售价门槛 | Input | 售价不得低于 成本×ratio |
| | 类目匹配检测 | Switch | 上架时校验类目是否正确 |
| | 图片合规检测 | Switch | 检测水印/Logo/版权元素 |
| | 库存透支保护 | Switch | 库存不足时阻止继续投流 |
| | 差评激增预警 | Switch | 差评数突增自动触发审批 |

> 风控 Agent 未开启时，其他 Agent 正常运行，无安全护栏。建议生产环境始终开启。

#### 3.3.3 依赖链路

```
店铺保活（必须） → 市场情报 → 商品上架 → 素材工厂 → 广告投放
                                   ↘ 定价策略
```

> 开通依赖未满足的 Agent 时，卡片会红字提示缺少的依赖，且不显示开通按钮。

#### 3.3.3 Agent 列表页 ✅

- **已开通区：** 已启用的 Agent 卡片（店铺保活标记红色「必须」Tag）
- **可开通区：** 按 5 层分组（基础管道/流量引擎/增值运营/支撑服务/独立服务），每层有色标
- **卡片信息：** Agent 名称、描述、风险等级 Tag、如需配置显示蓝色「策略配置」Tag、如需审批显示橙色「需要审批」Tag、依赖未满足显示红色警告
- 依赖满足后显示「开通」按钮

#### 3.3.4 Agent 详情页 ✅

- **页面 Header：** Agent 名称、描述、启用/停用开关（必须 Agent 置灰不可关）
- **运行统计：** 总执行次数、成功率、成功数、失败数
- **运行说明：** 功能描述、触发方式、风险等级、审批要求、常见失败原因、默认参数
- **策略配置：** 按 Agent 类型动态渲染对应配置项（定价规则/投流预算/SEO关键词+重新调研/人群标签+重新调研/CRM折扣+分群/退款额度+退货地址/素材尺寸+文案风格）
- **运行中任务：** 当前进行中/排队中的任务，每行可取消
- **任务日志：** 历史已完成/失败/已取消的任务，显示最近事件摘要
- **新建任务：** 店铺保活专用「新建会话检测」弹窗（选店铺+原因），其他Agent通用表单（任务名+目标+店铺），商品上架额外支持上传图片

### 3.4 模型中心（Model Center）

**定位：** AI 能力的"燃料"管理，决定 Agent 的聪明程度和运营成本。

#### 3.4.1 当前架构 ✅

模型中心完全重新设计，去掉了「供应商 → 模型」两级结构和费用统计，改为平铺的「Agent ↔ 模型」分配模式：

- **Agent 模型分配：** 每个 Agent 一行，右侧下拉框直接选择使用的模型。选项来自平台模型 + 我的模型中已激活的。
- **我的模型（自有模型）：** 用户点击「添加模型」→ 下拉选择模型类型（GPT-4o/Claude/DeepSeek/通义千问等 11 种）→ 输入 API Key → 自动创建。自有模型不统计费用。可删除。
- **月度用量统计：** 按月柱状图展示 Agent 调用/Token/浏览器会话/店铺数的月度趋势。

#### 3.4.2 平台预置模型 ✅

| 模型 | 特性 |
|------|------|
| 自动选择 | 由系统根据任务类型自动选择最合适的模型（默认） |
| Ark 电商专有模型 | 平台自主研发，专为电商运营场景优化的垂类模型 |
| GPT-4o | OpenAI 旗舰模型，适合高精度广告分析和财务对账 |
| GPT-4o Mini | OpenAI 轻量模型，速度快成本低，适合批量任务 |
| Claude Sonnet 4 | Anthropic 旗舰模型，擅长文案生成和内容审核 |
| DeepSeek-V3 | 高性价比模型，适合日常运营任务 |
| DeepSeek-R1 | 深度推理模型，适合复杂数据分析 |
| 通义千问 Max | 阿里云旗舰模型，中文理解能力强 |

#### 3.4.3 未来规划（P2）

- 模型使用统计（调用次数、Token 消耗趋势）
- 成本优化建议

### 3.7 财务台账（Billing）

**定位：** 商业化闭环的核心，查看账单、用量和费用明细。

#### 3.7.1 费用概览卡片 ✅
